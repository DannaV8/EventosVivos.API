# CLAUDE.md — EventosVivos.Infrastructure

## What this layer is

Everything that talks to the outside world: PostgreSQL via EF Core,
JWT generation, and dependency injection wiring.

Implements interfaces defined in `EventosVivos.Application`.
References: `EventosVivos.Domain` + `EventosVivos.Application`.

## Key design decisions

**`Event.Status` is computed — never mapped to a column.**
Only the private field `_cancelled` (bool) is persisted via `HasField("_cancelled")` in `EventConfiguration`.

**Optimistic concurrency on `Event`** via `RowVersion` (byte array, `IsRowVersion()`).
EF Core throws `DbUpdateConcurrencyException` on conflicting saves — the API middleware maps this to HTTP 409.
Never remove this — it's the only thing preventing overselling under concurrent load.

**All enums stored as strings** (`HasConversion<string>()`) — readable in the DB and safe to rename values.

**Venues are seeded reference data** in `AppDbContext.OnModelCreating` — they are never created from user actions.

**JWT claims**: `sub` (userId), `email`, `role` (`admin` | `user`). Read by `JwtService` from `IConfiguration` keys `Jwt:Key`, `Jwt:Issuer`, `Jwt:ExpiresInMinutes`.

---

## Indexes

- `Event`: `(VenueId, StartDateTime, EndDateTime)` — for venue overlap queries (RN-02)
- `Reservation`: `(EventId, Status)` — for fast capacity count queries
- `Reservation.ReservationCode`: unique, filtered on `IS NOT NULL`

---

## Hard rules

- Never add business logic here — that belongs in Domain or Application
- Never expose `AppDbContext` to Application layer — only through interfaces
- All DateTime values stored as UTC — Npgsql legacy timestamp behavior must be disabled:
  `AppContext.SetSwitch("Npgsql.EnableLegacyTimestampBehavior", false)`
- Use `HasConversion<string>()` for all enums
- User-facing error messages stay in Spanish; all identifiers and comments in English
